CocoaBench

An Evaluation Framework for General Agents with Compositional Cognitive Abilities

Paper GitHub Dataset

We introduce CocoaBench, a new benchmark designed to test the limits of Large Language Models in complex, multi-step reasoning tasks. Unlike previous benchmarks that focus on single-turn QA, CocoaBench simulates real-world software engineering environments.

Why CocoaBench?

Current evaluations often saturate quickly. CocoaBench introduces dynamic environment updates, requiring agents to not only generate code but also perceive changes in the state.

Evaluation Methodology

Our scoring mechanism combines multiple metrics to provide a holistic view of agent performance. The final score is computed as:

$$S = \frac{1}{N} \sum_{i=1}^{N} \left( \alpha \cdot \mathbb{1}[\text{pass}_i] + \beta \cdot \frac{T_{\text{ref}}}{T_i} + \gamma \cdot \text{efficiency}_i \right)$$

Where $\alpha$, $\beta$, and $\gamma$ are weighting factors, $\mathbb{1}[\text{pass}_i]$ is the pass indicator for task $i$, and $T_i$ represents the time taken.

Example Agent Loop

A typical agent interaction follows this pattern:

def agent_loop(task, environment):
    state = environment.reset()
    history = []
    
    while not state.is_terminal:
        # Observe current state
        observation = environment.observe()
        
        # Generate action via LLM
        action = llm.generate(
            task=task,
            observation=observation,
            history=history
        )
        
        # Execute and update
        state = environment.step(action)
        history.append((observation, action))
    
    return evaluate(state, task)
